= Pre-Crime Application Developer Documentation

== Overview

This documentation covers the architecture, development workflow, and operational
guidance for the Pre-Crime Application — a reference implementation of
**Domain-Driven Design (DDD)** and **Onion Architecture** using a modern
Kotlin/Spring Boot backend and an Angular frontend.
The code is only for educational purposes and is not meant to be production ready.

== Architecture

The project follows the **Onion Architecture** pattern:

* *Domain Ring* — Business logic, aggregates, value objects, domain events
* *Application Ring* — Orchestration services and event handlers
* *Infrastructure Ring* — REST controllers, persistence (jOOQ), messaging (Kafka)

== Quick Start

=== Prerequisites

* Java 25+
* Node.js 22+
* Docker & Docker Compose

=== Start Infrastructure

[source,bash]
----
cd docker
docker-compose up -d
----

=== Run Backend

[source,bash]
----
./mvnw spring-boot:run
----

=== Run Frontend

[source,bash]
----
cd ui
npm install
npm run generate-api
npm start
----

== Project Layout

* `src/` — Backend Kotlin source code
* `ui/` — Angular frontend
* `etc/developer-docs/` — This documentation
* `docker/` — Infrastructure (PostgreSQL, Kafka)
